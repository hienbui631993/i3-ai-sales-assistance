SOP: Configuring i3Ai Setup for optimal accuracy

Sample · illustrative example of the standardized i3Ai configuration SOP
A sample of the standardized i3 configuration SOP — the worked example of the format Phase 2 runs on: configure → verify → monitor weekly → audit monthly → improve. That loop is what keeps the delivered solution performing as the contract promised. (Company, contacts and images below are illustrative samples.)

Objective

Establish a consistent process for configuring and verifying i3Ai detection settings on all cameras to ensure proper operation and detection capabilities at Northwind Retail's sites. Regular checks and detailed documentation prevent future issues and enhance overall performance.

Scope

Applies to all support staff responsible for configuring and maintaining i3Ai cameras across all Northwind Retail sites.

Limitations

Environmental and infrastructure conditions outside i3's control — network performance, power fluctuations, camera lens/dome cleanliness, severe weather and other acts of God — may affect i3Ai accuracy and performance; i3 makes no accuracy claims under those circumstances.

Procedure · 1 — Infrastructure & configuration requirements

Server: i3Ai Server with GPU is required — GPU = GTX1050i (minimum), GTX1060 or better (recommended).

Software (all packages from i3's Downloads site, i3international.com/download; the latest packages are pre-staged on the local D: partition of the GPU-enabled site servers, ready for installation):

  • Install the latest SRX-Software (v7.3.2.120 as of June 19, 2024) on each server.
  • Install the NVIDIA driver (64-bit) on the i3Ai Server (GPU-enabled server).
  • Install the latest i3Ai Server software (v7.3.2.90 as of June 19, 2024).

i3Ai Setup camera configuration requirements (installation best practices — camera angle, bitrate, maximum coverage area — provided by i3 Team Leads due June 28, 2024):

  • Minimum acceptable resolution: 1280×960 (1.3 MP).
  • Required i3 and Axis IP camera frame rate: 5 FPS.
  • Required HR16v2 encoder frame rate: 6 FPS.
  • "Max detection area" setting must be enabled.
  • Max detection area must be larger than the detection area.
  • Object detection must be set to "Head" and "Person".
  • Object count must be set to ≥ 1, Duration must be set to ≥ 1.

Procedure · 2 — Verification

  • Post-configuration checks: verify each camera's frame-rate settings and confirm the maximum-detection settings are applied.
  • Testing: run an AI detection + alert test at night wherever possible; if not, temporarily set Alarm Active Time to 24/7 for the test, then return it to 10PM–7AM. Review results for accuracy and reliability.

Procedure · 3 — Ongoing monitoring & maintenance

  • Weekly checks that configurations remain consistent; recalibrate if discrepancies are found.
  • Incident response: an undetected event triggers an immediate camera-settings review; adjust as necessary and document the changes.

Procedure · 4 — Documentation & reporting

  • Maintain detailed logs of all configurations and changes, with issues encountered and resolutions applied.
  • Monthly audits of camera settings; audit reports go to the Technical Support Manager.

Procedure · 5 — Continuous improvement

  • Collect feedback from on-site teams on AI performance; adjust and update configurations to feedback and new standards.

Troubleshooting common issues

  • Frame-rate discrepancies: check the network bandwidth supports the configured rates; verify camera firmware is up to date.
  • Detection failures: ensure the lens is clean and unobstructed; review recent changes to AI algorithms or software updates.
