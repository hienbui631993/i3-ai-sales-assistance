How Vision is put together

Vision is an agentic sales assistant that lives inside i3Host (never public). For v1, i3 reps and integrators access it through i3Host. (Channel-partner access and partner self-host are a future consideration — not decided yet.) This is the build map — top to bottom, who talks to what.

1 · Access tiers (who can use Vision)
🧑‍💼i3 Sales Reps
Inside i3Host. Every stage / agent.
Full access · v1
🔧Integrators
Linked into i3Host; run Vision on their own deals.
Full access · v1
🤝Channel Partners
A possible later direction — a scoped subset of agents. Not decided yet.
Future · TBD
🏠Partner self-host
Partners running their own instance — possible later. Not decided yet.
Future · TBD
▼   authenticated via i3Host SSO + role scoping
2 · Host application
🖥️i3Host (web app)
The UI Vision runs inside — quoting tools, calls, Teams & GoToMeeting transcripts. Hosts the Vision UI and enforces who sees which agent.
🔐Identity & access
SSO + role (rep / integrator). Role decides agent visibility.
▼
3 · Vision orchestrator + reasoning
🧠Vision Orchestrator
Detects the sales stage, picks the agent, enforces stage rules, and decides what runs next. Rules control progression; the LLM guides quality.
✨LLM (function calling)
OpenAI or Claude. Same architecture either way — system prompt + tools + retrieval. Keep tool definitions provider-agnostic.
▼   orchestrator invokes the right agent
